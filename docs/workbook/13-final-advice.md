## 복잡도는 절대 흉내낼 수 있는 스펙이 아닙니다
대규모 클라우드 시스템을 가져다 붙이는 것은 누구나 돈과 시간이 있으면 할 수 있습니다. 
하지만 아무것도 없는 로컬 머신 하나 위에서 **"카메라 인풋 👉 얼굴 분류 모델 👉 비동기 스레드 파이프라인 👉 실시간 SSE 푸싱 👉 Angular 캔버스 SVG 렌더링"** 이라는 완결된 풀스택(Full-Stack) 사이클을 구축하는 일은, 파편화된 기술을 하나의 예술품처럼 엮어내는 치밀한 설계 감각을 증명합니다.

## 시스템 아키텍트의 시선으로 코드를 읽으십시오
- 왜 쿼리문을 안 치고 그냥 JSON 객체 덮어쓰기를 선택했는가. (효율 극대화)
- 상태 렌더링에 왜 툭하면 리렌더링 되는 React 대신 Angular 스택을 택했나. (오버레이 퍼포먼스 제어)
- 왜 OpenCV 로 처리된 사진 위에 박스를 직접 안 그리고 프론트로 이벤트를 던져 SVG를 따로 그렸나. (투명하고 미려한 디자인을 위해서)

본 워크북을 다 숙지하셨다면 과감히 코드를 부러뜨리고(`yarn start` & `app.py`) 에러와 직접 마주해 보시길 권장합니다.
