# [05] 프론트엔드 심화 해부: Angular와 Sci-Fi UI 디자인

프론트엔드 레포지토리는 BCC 시스템의 꽃입니다. 데이터가 아무리 정교해도 보여주는 방식이 투박하면 가치가 떨어집니다.
자바스크립트 템플릿(Jinja 등)을 제거하고, 엔터프라이즈 모던 웹앱의 표준인 **Angular (v15+)** 를 채택했습니다.

## 1. 왜 컴포넌트(Component) 기반 디자인인가?
BCC 화면은 대략 3가지 구역으로 나뉩니다.
- **스트림 레이어 패널**: 비디오와 오버레이 SVG 애니메이션
- **제어 커맨드 패널**: 인물을 등록하고 삭제하는 오퍼레이팅 액션
- **글로벌 로그 패널**: 시스템 메세지를 띄워주는 터미널 텍스트

Angular는 이들 조각들을 각각 분리된 부품(`*.component.ts`)으로 쪼개서 조립하는 데 강력한 성능을 발휘합니다. 특정 컴포넌트가 다운되더라도 다른 요소들은 정상 렌더링 됩니다.

## 2. Sci-Fi HUD 디자인 리소스: SCSS 순수 구현
테일윈드(Tailwind)와 같은 범용 유틸리티 CSS를 버리고, 우리가 통제 가능한 **Custom SCSS** 기반의 코드를 선택했습니다.
모든 사이버 디스플레이 효과는 아래 기법에 의존합니다.

- **Neon Glow (글로우 효과)**: `box-shadow`와 `text-shadow`를 겹쳐서 블러 처리하여 발광하는 LED 느낌 구현.
- **Color Palette (컬러톤)**: RGB 기반이 아닌 HSL 기반 (에너지 넘치는 시안 파랑 `#00f3ff`, 강력한 노랑 `#f9ca24`) 사용.
- **Typography (폰트)**: 완벽한 이질감 없는 몰입을 위해 구글 웹폰트 `Noto Sans KR`, 그리고 굵고 압도적인 가독성을 내는 `Black Han Sans`를 CSS에 수입(import)하여 시스템 글꼴을 전부 교체했습니다.

## 3. RxJS 기반의 비동기 스트리밍
SSE 연결망에서 초당 최대 수십 개의 이벤트가 튀어나옵니다. Angular는 내장된 `RxJS` 를 활용하여 이 이벤트들을 물 흐르듯 가로채서(Subscribing), 과도하게 중복 수신되는 것을 필터링하거나 지연 렌더링 하여 화면 끊김 성능(Jank)을 방어합니다.
