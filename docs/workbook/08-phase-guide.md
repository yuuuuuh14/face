# [08] 프로젝트 에볼루션: 페이즈별 회고 가이드

BIOMETRIC_CONTROL_CENTER(BCC)가 v3.5 Stable 버전에 도달하기까지, 개발 단계(Phase)는 파편화된 기술들을 하나로 융합하는 여정이었습니다.

## Phase 1: 로컬로 회귀하라 (The Great Escape)
- **개요**: 처음 이 아이를 기획했을 땐 최신 유행인 Docker 컨테이너 여러 개로 모듈을 쪼갰습니다만, 결국 카메라 패스스루 통로를 뚫고 X11 디스플레이를 포워딩하다 진이 빠졌습니다.
- **성취**: 백엔드를 전부 로컬 네이티브 환경(Python Venv 기반 `app.py`)으로 내렸고, 이로 인해 개발 생산성이 **10배** 이상 오르고 OpenCV에서 MJPEG를 직접 스트리밍 하는 원시적인 구조로 랙을 최소화했습니다. 

## Phase 2: '나'를 알아봐달라 (Identity Persistence)
- **개요**: 화면에 네모 박스는 잘 그렸지만 시스템은 사람을 구별할 줄 몰랐습니다. "인물 등록" 기능을 도입해야 했습니다.
- **성취**: 무거운 PostgreSQL 서버 대신 완전 파일 기반의 내장 `SQLite` DB를 채택했습니다. `faces.db` 에 얼굴 벡터의 직렬화 데이터를 우겨넣고 메모리에 캐시하여 코사인 매칭을 수행함으로써, 무결성과 스피드를 동시에 확보했습니다.

## Phase 3: 멋이 곧 전부다 (Sci-Fi Cybernetic UI)
- **개요**: 기존의 프론트엔드 버튼과 화면은 너무 은행/쇼핑몰 같았습니다. 우리는 영화 속 비밀 요원의 '그것'을 원했습니다.
- **성취**: 자바스크립트를 전면 Angular 워크스페이스 형태로 뜯어 고쳤습니다. Tailwind의 유틸성을 버리고 하드코딩된 SCSS 속성을 갈아 넣어 네온 글로우 스캔라인 효과와 Material 다이얼로그 모달을 장착했습니다. 

## Phase 4 (현재 진행): 딥 인텔리전스 레이어
- **성취**: 단순한 매칭을 넘어, InsightFace 에서 추가적으로 스쿼드 속성(Gender, Age)을 뽑아내도록 파이프라인을 우회/추가 확장했습니다. 현재는 무의미한 에러를 뿜는 메모리 누수를 잡고, 파이썬 기반 비동기 스레드 모니터링 체계를 고도화하기 위한 리팩토링이 진행중입니다.
